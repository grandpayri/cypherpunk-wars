Technical Architecture: The Decentralized Bunker

1. Frontend: Agnostic SPA Hosting

CypherPunk Wars is built as a Single Page Application (SPA). While our Kaspathon demo is hosted on GitHub Pages, the architecture is compatible with any Node.js web server or decentralized hosting (IPFS/Arweave).

  • Zero Backend: There is no proprietary database or server-side logic.
  • Portability: Any operator can host their own "Bunker Interface" by simply serving the static JS/WASM files.

2. Execution: Client-Side Web3 WASM

The game logic is executed entirely Client-Side using a Web3-compatible browser environment.

  • WASM Engine: High-performance cryptographic functions and key derivations are handled by the Kaspa WASM32 SDK.
  • Sovereign Logic: Turns are processed locally and broadcast as authenticated proofs to the DAG.

3. Data Layer: The BlockDAG as a Database

CypherPunk Wars treats the Kaspa BlockDAG as its primary persistent storage engine.

  • State Management: Player metadata, War Chest ($PUNKW) balances, and Sector claims are stored on-chain.
  • Pruning Resistance: Vital game state is maintained through a combination of UTXO metadata and permanent transaction records.
